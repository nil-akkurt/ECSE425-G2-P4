**RISC-V Pipelined Processor (VHDL)**

This project implements a 5-stage pipelined RISC-V processor in VHDL, following the standard instruction pipeline of Instruction Fetch (IF), Instruction Decode (ID), Execute (EX), Memory (MEM), and Write Back (WB). The design supports a subset of the RISC-V instruction set, including arithmetic, logical, shift, comparison, memory access, and control flow instructions. The processor is structured in a modular way, with separate VHDL files for key datapath components such as the program counter, instruction memory, register file, immediate generator, ALU, and data memory, as well as control logic modules responsible for instruction decoding, ALU control, and hazard detection. Pipeline registers are used between each stage to store intermediate values and enable proper pipelined execution.
The testbench generates register_file.txt, containing the final values of the 32 registers, and memory.txt, containing the contents of data memory. These outputs are used to verify the correctness of the processor’s execution against expected results.

The design assumes that the program counter is initialized to zero and that hazards are handled through stalling using a dedicated hazard detection unit, as forwarding is not required for full credit. Overall, the project focuses on building a form of a modular implementation of a pipelined RISC-V processor, with emphasis on correct datapath design, control signal generation, and integration with the provided testing framework.
